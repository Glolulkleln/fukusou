这是一份为您整理好的完整数据库数据字典（Data Dictionary）。它完全对应我们之前在 SQL Server 中创建的 8 张核心数据表。

在实际的软件工程交付或向甲方汇报时，这份字典是极其关键的验收文档，它清晰定义了底层数据的流转规则和约束规范。

### 1. 用户表 (`user`)

用于存储 C 端小程序用户的基本授权信息与状态。

| 字段名 | 数据类型 | 长度 | 主键/约束 | 默认值 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| `id` | INT | - | 主键, 自增 | - | 用户唯一标识 |
| `openid` | NVARCHAR | 100 | 唯一 | - | 微信用户唯一标识 |
| `nickname` | NVARCHAR | 100 | - | - | 微信昵称 |
| `avatar_url` | NVARCHAR | 255 | - | - | 头像图片链接 |
| `phone` | NVARCHAR | 20 | - | - | 手机号码 |
| `status` | TINYINT | - | - | `1` | 账号状态：`1`=正常, `0`=黑名单禁用 |
| `created_at` | DATETIME | - | - | `GETDATE()` | 首次注册/授权时间 |

---

### 2. 服装分类表 (`category`)

用于首页与分类页的左侧导航动态管理。

| 字段名 | 数据类型 | 长度 | 主键/约束 | 默认值 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| `id` | INT | - | 主键, 自增 | - | 分类唯一标识 |
| `name` | NVARCHAR | 50 | 非空 | - | 分类名称（如：面试正装） |
| `sort_order` | INT | - | - | `0` | 排序权重（值越小越靠前） |

---

### 3. 服装商品表 (`clothing`)

存储租赁商品的核心信息，规格采用 JSON 字符串以支持动态尺码扩展。

| 字段名 | 数据类型 | 长度 | 主键/约束 | 默认值 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| `id` | INT | - | 主键, 自增 | - | 服装唯一标识 |
| `category_id` | INT | - | 非空 | - | 关联所属分类ID |
| `name` | NVARCHAR | 100 | 非空 | - | 服装商品名称 |
| `main_image` | NVARCHAR | 255 | 非空 | - | 列表主图/封面图链接 |
| `specs` | NVARCHAR | MAX | 非空 | - | 规格尺码列表（存储为JSON字符串） |
| `rent_price` | DECIMAL | 10,2 | 非空 | - | 每日租金金额 |
| `deposit_amount` | DECIMAL | 10,2 | 非空 | - | 租赁所需押金金额 |
| `status` | TINYINT | - | - | `1` | 上下架状态：`1`=上架, `0`=下架 |
| `created_at` | DATETIME | - | - | `GETDATE()` | 商品录入时间 |

---

### 4. 租赁订单表 (`orders`)

整个系统的核心中枢，控制着物流、归还和押金退还的时间线。

| 字段名 | 数据类型 | 长度 | 主键/约束 | 默认值 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| `id` | INT | - | 主键, 自增 | - | 内部订单ID |
| `order_no` | NVARCHAR | 50 | 唯一, 非空 | - | 面向用户的业务订单号 |
| `user_id` | INT | - | 非空 | - | 关联下单用户ID |
| `clothing_id` | INT | - | 非空 | - | 关联租赁服装ID |
| `selected_spec` | NVARCHAR | 100 | 非空 | - | 用户下单时选择的具体规格 |
| `rent_start_time` | DATETIME | - | 非空 | - | 租赁开始时间 |
| `rent_end_time` | DATETIME | - | 非空 | - | 预计归还截止时间 |
| `actual_return_time` | DATETIME | - | 允许为空 | - | 实际确认归还时间 |
| `rent_amount` | DECIMAL | 10,2 | 非空 | - | 租金总额（单价×天数） |
| `deposit_amount` | DECIMAL | 10,2 | 非空 | - | 冻结/支付的押金金额 |
| `total_amount` | DECIMAL | 10,2 | 非空 | - | 实际支付总金额（租金+押金） |
| `status` | TINYINT | - | - | `0` | 订单状态：`0`=待支付, `1`=待发货, `2`=租赁中, `3`=待归还, `4`=已完成, `5`=已取消 |
| `express_no` | NVARCHAR | 100 | 允许为空 | - | 发货物流快递单号 |
| `created_at` | DATETIME | - | - | `GETDATE()` | 订单生成时间 |

---

### 5. 押金资金流水表 (`deposit_flow`)

严格记录每一笔押金的变动，确保账实相符与售后追溯。

| 字段名 | 数据类型 | 长度 | 主键/约束 | 默认值 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| `id` | INT | - | 主键, 自增 | - | 流水记录唯一标识 |
| `order_no` | NVARCHAR | 50 | 非空 | - | 关联的订单号 |
| `user_id` | INT | - | 非空 | - | 关联用户ID |
| `amount` | DECIMAL | 10,2 | 非空 | - | 发生变动的金额 |
| `flow_type` | TINYINT | - | 非空 | - | 资金变动类型：`1`=用户支付, `2`=系统退还, `3`=损坏/逾期扣除 |
| `status` | TINYINT | - | - | `0` | 处理状态：`0`=处理中, `1`=已成功, `2`=失败/驳回 |
| `remark` | NVARCHAR | 255 | 允许为空 | - | 变动原因备注（如：衣服破损扣款50元） |
| `created_at` | DATETIME | - | - | `GETDATE()` | 流水发生时间 |

---

### 6. 用户评价与买家秀表 (`review`)

用于构建平台信任护城河，关联服装详情页展示。

| 字段名 | 数据类型 | 长度 | 主键/约束 | 默认值 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| `id` | INT | - | 主键, 自增 | - | 评价唯一标识 |
| `order_no` | NVARCHAR | 50 | 非空 | - | 关联订单号（限制一单评一次） |
| `user_id` | INT | - | 非空 | - | 评价用户ID |
| `clothing_id` | INT | - | 非空 | - | 被评价服装ID |
| `rating` | TINYINT | - | 非空 | - | 星级评分（1至5星） |
| `content` | NVARCHAR | MAX | 允许为空 | - | 纯文本评价内容 |
| `images` | NVARCHAR | MAX | 允许为空 | - | 买家秀图片列表（存储为JSON字符串） |
| `created_at` | DATETIME | - | - | `GETDATE()` | 评价提交时间 |

---

### 7. 首页轮播活动表 (`banner`)

控制小程序首页顶部的活动展示与跳转。

| 字段名 | 数据类型 | 长度 | 主键/约束 | 默认值 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| `id` | INT | - | 主键, 自增 | - | 轮播图标识 |
| `image_url` | NVARCHAR | 255 | 非空 | - | 图片网络存储地址 |
| `target_link` | NVARCHAR | 255 | 允许为空 | - | 点击图片后的跳转路径参数 |
| `sort_order` | INT | - | - | `0` | 轮播顺序 |
| `status` | TINYINT | - | - | `1` | 状态：`1`=展示中, `0`=已隐藏 |

---

### 8. B端后台管理员表 (`admin`)

控制 PC 端后台系统的登录与精细化权限分配。

| 字段名 | 数据类型 | 长度 | 主键/约束 | 默认值 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| `id` | INT | - | 主键, 自增 | - | 管理员标识 |
| `username` | NVARCHAR | 50 | 唯一, 非空 | - | 后台登录账号 |
| `password_hash` | NVARCHAR | 255 | 非空 | - | 登录密码（需经过加密哈希处理） |
| `role` | TINYINT | - | 非空 | - | 角色权限：`1`=超级管理员, `2`=校区运营, `3`=仓库入库 |
| `phone` | NVARCHAR | 20 | 允许为空 | - | 管理员联系方式 |
| `status` | TINYINT | - | - | `1` | 账号状态：`1`=正常, `0`=离职/禁用 |
